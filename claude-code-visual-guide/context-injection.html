Type 1

                
Instruction Reminders

                

                  

                    When:
                    Every turn
                  

                  

                    Tokens:
                    ~300
                  

                  

                    Purpose:
                    General behavioral guidelines — always present
                  

                

              

              

                
Example content

                
<system-reminder>
You are Claude Code, Anthropic's official CLI for Claude.
You must always use the Task tools to track work.
Never make up tool results. Always confirm destructive
operations before proceeding.
</system-reminder>

Type 2

                
Plugin / Hook Context

                

                  

                    When:
                    Turn 1 only (SessionStart hook)
                  

                  

                    Tokens:
                    ~1,200
                  

                  

                    Purpose:
                    Skill bootstrap, plugin instructions
                  

                

              

              

                
Example content

                
<system-reminder>
SessionStart hook additional context:
<EXTREMELY_IMPORTANT>
You have superpowers.
[using-superpowers skill content...]
You MUST use the Skill tool before attempting
any task that has a matching skill.
</EXTREMELY_IMPORTANT>
</system-reminder>

Core Design

          
The Cache-Friendly Pattern

          

            Why dynamic context goes into messages instead of the system prompt —
            and how that decision saves thousands of tokens per turn.
          

        


        

          

            Freeze the system prompt, inject dynamics through messages.
          


          

            The system prompt carries cache_control: ephemeral. Any change — even a single
            character — invalidates the cache and forces a full re-tokenization on the next API call.
            Dynamic context like the current date, CLAUDE.md content, or session hook output changes
            every turn, so it lives in the user message instead.
          


          

            

              
Before — breaks cache every turn

              
# system changes → cache miss every turn
system = [{
  "text": f"Today is {date}. You are a coding assistant...",
  "cache_control": "ephemeral"  # ← changes!
}]

            

            

              
After — cache-friendly

              
# system frozen → cache hits every turn
system = [{
  "text": "You are a coding assistant...",
  "cache_control": "ephemeral"  # ← stable!
}]
messages = [{
  "role": "user",
  "content": [
    {"text": f"<system-reminder>Today is {date}</system-reminder>"},
    {"text": user_message}
  ]
}]

Chapter 9 — Model Context Protocol

MCP Servers

External tools and data sources via the Model Context Protocol — an open standard for connecting AI models to services like Playwright, Slack, Jira, CircleCI and more.

Session Startup

Tool Discovery

When Claude Code starts, it runs an 80+ API call initialization sequence to connect to MCP servers, validate each tool's JSON schema, and generate human-readable descriptions.

Token Optimization

Deferred Loading

When many MCP tools are configured, sending all full schemas on every API call wastes ~20,000 tokens per turn. Deferred loading solves this by only including tool names upfront, loading full schemas on demand via ToolSearch.

On-demand loading

ToolSearch — Loading a Schema on Demand

When Claude needs to use an MCP tool, it first calls ToolSearch to fetch the full schema, then calls the actual tool.

ToolSearch call 
{
  "name": "ToolSearch",
  "input": {
    "query": "select:mcp__claude_ai_Slack__slack_send_message",
    "max_results": 1
  }
}
Result — full schema now available 
{
  "name": "mcp__claude_ai_Slack__slack_send_message",
  "description": "Send a message to a Slack channel...",
  "input_schema": {
    "type": "object",
    "properties": {
      "channel_id": { "type": "string" },
      "text": { "type": "string" },
      "thread_ts": { "type": "string" }
    },
    "required": ["channel_id", "text"]
  }
}
Tool Naming

Naming Convention

Every MCP tool name follows a consistent three-part pattern using double-underscore separators, making it easy to identify the originating server from any tool name.

mcp __ claude_ai_Slack __ slack_send_message
Prefix mcp Always "mcp" — identifies this as an MCP tool vs built-in or plugin tool
Server name claude_ai_Slack The MCP server identifier, taken from the server configuration
Tool name slack_send_message The specific capability exposed by that server
mcp __ playwright __ browser_click
mcp __ circleci-mcp-server __ get_build_failure_logs
mcp __ claude_ai_Slack __ slack_send_message
mcp __ claude_ai_Atlassian_JIRA_Confluence __ getJiraIssue
System context

MCP Server Instructions

Each MCP server provides usage instructions that are injected into the session as a <system-reminder> block, giving Claude context about how to use that server's tools effectively.

System prompt injection (system-reminder) 
<system-reminder>
## claude.ai Amplitude
Amplitude is a product analytics platform. Most tools require a
projectId (appId). At the start of a session, call 'get_context'
to see the user's organization and available projects.

## claude.ai Slack
Use channel IDs (not names) for all channel operations. Call
slack_search_channels to find a channel ID from a name.

## circleci-mcp-server
Use get_latest_pipeline_status to check the current state of a
pipeline before triggering a rerun.
</system-reminder>

Configuration

How Local Servers Are Configured

Local MCP servers are defined as gateway scripts. Claude Code runs these scripts to spawn the MCP server process, then communicates via stdio.

~/.claude/mcp-gateway-playwright.sh 
#!/bin/bash
npx @anthropic-ai/mcp-playwright
The gateway script is a thin wrapper that launches the actual MCP server. Claude Code discovers it, executes it, and then communicates over stdio using the MCP protocol to enumerate tools and handle calls.
Local servers run as child processes on the same machine. They have access to the local filesystem, browser, and any locally installed tools.
Cloud servers (Amplitude, Slack, Jira) run remotely as claude.ai integrations. They authenticate via OAuth and proxy calls to the respective external APIs.
Startup cost: Each configured MCP server adds validation + description calls on startup. With 6 servers totaling 39 tools, that is 78 initialization calls before the first user message is processed.
MCP Resources

Resources vs Tools

Beyond tools, MCP servers can expose resources — structured data objects the model can read directly. Two built-in tools provide access to MCP resources.

ListMcpResourcesTool
Enumerate all resources exposed by connected MCP servers. Returns a list of resource URIs and their metadata (name, MIME type, description).
Call 
{
  "name": "ListMcpResourcesTool",
  "input": {}
}
ReadMcpResourceTool
Read the contents of a specific MCP resource by URI. Returns the resource content — which may be text, JSON, or binary data depending on the server.
Call 
{
  "name": "ReadMcpResourceTool",
  "input": {
    "server_name": "playwright",
    "uri": "playwright://docs/api"
  }
}
Resources are distinct from tools: tools perform actions (click, send message, run pipeline), while resources expose data that can be read passively (documentation, configuration, reference data). Not all MCP servers expose resources — most only expose tools.

Section 3

          
SessionStart Hook — Deep Dive

          
The session-start script performs five steps every time a session begins. Click any step to see the real code.

        


        


          
          

            
            

              

                
Checks whether ~/.config/superpowers/skills exists — the legacy pre-plugin path. If found, outputs a deprecation warning so the user knows to migrate.

                

                  
session-start (bash)

                  
# Check for legacy skills directory
if [ -d "${HOME}/.config/superpowers/skills" ]; then
  echo "WARNING: Legacy skills dir detected at ~/.config/superpowers/skills" >&2
  echo "Please migrate to the plugin system." >&2
fi

                

              

            

          


          
          

            
            

              

                
Reads the using-superpowers/SKILL.md file from the plugin's skills directory. This is the bootstrapping skill that tells Claude how to use all other skills in the plugin.

                

                  
session-start (bash)

                  
# PLUGIN_ROOT is set by Claude Code to the plugin's cache dir
SKILL_FILE="${CLAUDE_PLUGIN_ROOT}/skills/using-superpowers/SKILL.md"

if [ ! -f "${SKILL_FILE}" ]; then
  echo "ERROR: Skill file not found: ${SKILL_FILE}" >&2
  exit 1
fi

skill_content=$(cat "${SKILL_FILE}")

                

              

            

          


          
          

            
            

              

                
The skill file is arbitrary Markdown. Before embedding it inside a JSON string it must be escaped: backslashes doubled, double-quotes escaped, and newlines replaced with \n.

                

                  
session-start (bash)

                  
escape_for_json() {
  local input="$1"
  local output

  # 1. Escape backslashes first (must be first!)
  output="${input//\\/\\\\}"

  # 2. Escape double quotes
  output="${output//\"/\\\"}"

  # 3. Replace literal newlines with \n
  output="${output//$'\n'/\\n}"

  printf '%s' "$output"
}

session_context=$(escape_for_json "${skill_content}")

                

              

            

          


          
          

            
            

              

                
The escaped content is wrapped in <EXTREMELY_IMPORTANT> XML tags. This signals to Claude that the injected context should be treated as high-priority system instructions.

                

                  
session-start (bash)

                  
context_wrapped="<EXTREMELY_IMPORTANT>\\n${session_context}\\n</EXTREMELY_IMPORTANT>"

# Re-escape the wrapper (the tags themselves are safe,
# but session_context was already escaped above)
final_context="${context_wrapped}"

                

              

            

          


          
          

            
            

              

                
The output format differs by platform. Claude Code (native) uses hookSpecificOutput; Cursor uses additional_context. The script detects the platform via environment variables.

                

                  
session-start (bash)

                  
if [ -n "${CURSOR_PLUGIN_ROOT:-}" ]; then
  # Cursor format
  printf '{"additional_context": "%s"}\n' "$final_context"

elif [ -n "${CLAUDE_PLUGIN_ROOT:-}" ]; then
  # Claude Code native format
  printf '{"hookSpecificOutput": {"hookEventName": "SessionStart", "additionalContext": "%s"}}\n' \
    "$final_context"

else
  # Unknown platform — output nothing
  exit 0
fi

The Polyglot Runner

          

            run-hook.cmd is a single file that executes correctly on both Windows (as a .cmd batch file) and Unix (as a shell script), using the polyglot comment trick.
          

          

            

              run-hook.cmd
            

            
: << 'CMDBLOCK'
@echo off
REM Windows: finds bash.exe on PATH and runs the named script
SET SCRIPT_NAME=%1
FOR /F "delims=" %%i IN ('where bash 2^>nul') DO SET BASH=%%i
"%BASH%" "%~dp0%SCRIPT_NAME%"
goto :eof
CMDBLOCK
# Unix: the CMDBLOCK heredoc was a no-op; run the script directly
SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
SCRIPT_NAME="$1"
exec bash "${SCRIPT_DIR}/${SCRIPT_NAME}" "$@"

          

        


      

    


    
    

      


        

          
Section 5

          
What a Plugin Can Add

          
A plugin package can contribute four types of components, each living in its own subdirectory and registered automatically at startup.

        


        


          
          

            

              
            

            
Hooks

            
Lifecycle callbacks fired at session start, before/after tool execution, and on prompt submission. Can inject context or block actions.

            
hooks/hooks.json

          


          
          

            

              
            

            
Skills

            
Behavioral workflow modules stored as Markdown files. Loaded on demand via the Skill tool — zero token cost until needed.

            
skills/<skill-name>/

          


          
          

            

              
            

            
Agents

            
Custom agent type definitions with name, description, model, and system prompt. Registered as sub-agents available to the Agent tool.

            
agents/<name>.md

          


          
          

            

              
            

            
Commands

            
Slash commands that appear in the Claude Code command palette. Defined in Markdown with a YAML front-matter description.

            
commands/<name>.md

          


        

      

    


    
    

      


        

          
Section 6

          
Configuration Reference

          
The key configuration files that define and enable a plugin.

        


        


          

            
            
            
            
          


          
          

            
The plugin manifest lives at .claude-plugin/plugin.json and declares the plugin's identity.

            

              

                json
              

              
{
  "name": "superpowers",
  "description": "Core skills library for Claude Code",
  "version": "5.0.5",
  "author": {
    "name": "Jesse Vincent",
    "email": "jesse@fsck.com"
  },
  "homepage": "https://github.com/obra/superpowers",
  "license": "MIT",
  "keywords": ["skills", "tdd", "debugging", "collaboration"]
}

            

          


          
          

            
Hook registrations live at hooks/hooks.json. The matcher field is a regex that gates when the hook fires.

            

              

                json
              

              
{
  "hooks": {
    "SessionStart": [
      {
        "matcher": "startup|clear|compact",
        "hooks": [
          {
            "type": "command",
            "command": "\"${CLAUDE_PLUGIN_ROOT}/hooks/run-hook.cmd\" session-start",
            "async": false
          }
        ]
      }
    ],
    "PreToolUse": [
      {
        "matcher": ".*",
        "hooks": [
          {
            "type": "command",
            "command": "\"${CLAUDE_PLUGIN_ROOT}/hooks/run-hook.cmd\" pre-tool-use",
            "async": true
          }
        ]
      }
    ]
  }
}

            

          


          
          

            
Hooks communicate back to the host via stdout JSON. The format differs by platform.

            
Claude Code (native)

            

              

                json
              

              
{
  "hookSpecificOutput": {
    "hookEventName": "SessionStart",
    "additionalContext": "<EXTREMELY_IMPORTANT>\n...skill content...\n</EXTREMELY_IMPORTANT>"
  }
}

            

            
Cursor

            

              

                json
              

              
{
  "additional_context": "<EXTREMELY_IMPORTANT>\n...skill content...\n</EXTREMELY_IMPORTANT>"
}

            

          


          
          

            
Plugins are activated in ~/.claude/settings.json. The key is pluginName@marketplace.

            
~/.claude/settings.json (global)

            

              

                json
              

              
{
  "enabledPlugins": {
    "superpowers@claude-plugins-official": true
  },
  "voiceEnabled": true,
  "skipDangerousModePermissionPrompt": true,
  "model": "opus[1m]"
}

            

            
~/.claude/settings.local.json (local overrides)

            

              

                json
              

              
{
  "permissions": {
    "allow": [
      "mcp__playwright__browser_snapshot",
      "Bash"
    ]
  }
}

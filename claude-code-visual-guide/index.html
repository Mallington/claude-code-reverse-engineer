Architecture

Four Layers, One Request

Every Claude Code response passes through these four layers. Click any component to explore it in depth.

Layer 1 Extensions & Plugins
Plugins Custom hooks and lifecycle callbacks Skills Reusable instruction sets from .claude/skills/ Hooks PreToolUse · PostToolUse · Stop events MCP Servers Model Context Protocol — custom tools & resources
Layer 2 Context Assembly
System Prompt ~8K tokens · prompt-cached · injected once Tool Definitions 57 built-in tools as JSON schemas System Reminders Per-turn injected context and environment data Environment Context cwd · git status · repo context Message History Conversation history · compaction · window tracking
Layer 3 API Request
model e.g. claude-sonnet-4-5 · claude-opus-4 max_tokens Default 32K · configurable per request system Full assembled system prompt · prompt-cached blocks tools 57+ JSON-schema tool definitions messages user · assistant · tool_result turns stream SSE streaming · token-by-token output
Layer 4 Response Handling
Text Blocks Streamed content · rendered to terminal Tool Use Triggers local tool execution · parallel supported Tool Result Execution output fed back · next API turn Sub-Agents Parallel task execution · isolated contexts Agentic Loop Continues until end_turn or max_turns limit Persistence Transcript storage · resumable sessions

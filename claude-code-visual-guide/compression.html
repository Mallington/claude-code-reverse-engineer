Chapter 9 · Context Management

Context Compression
System

Four layered strategies — from zero-cost server edits to full LLM summarization — that keep long-running sessions productive while minimizing API spend.

Architecture

Four-Layer Pipeline

Each layer fires at a different threshold and has different cost tradeoffs. Click any layer to expand details. Session-Memory Compact is tried before Full LLM Compact because it is free.

Compression Pipeline — Every Turn
L1
API Microcompact apiMicrocompact.ts
Trigger: every request  ·  server-side context_management edits
Zero Cost ant-only

Generates a ContextManagementConfig object sent with every API request. The server applies these edits before processing — the client never pays for cleared tokens.

Strategy 1
clear_thinking_20251015
Keeps all thinking blocks by default. When idle >1h + cache miss: keeps only last 1 thinking turn. Cannot clear ALL thinking (API requires value ≥ 1).
Strategy 2
clear_tool_uses_20250919
Fires when input_tokens > 180K. Clears until ~40K tokens of tool results remain. Clearable: Shell, Glob, Grep, Read, WebFetch, WebSearch, Edit, Write.
Trigger
Input tokens > 180,000 (DEFAULT_MAX_INPUT_TOKENS)
Target
~40,000 tokens of tool results remain
L2
Client Microcompact microCompact.ts
Trigger: every request  ·  client-side tool result clearing
Zero Cost

Two mutually exclusive paths. Time-based path fires when gap since last assistant message exceeds 60 minutes (matching server cache TTL). Cached-MC path (ant-only) uses cache_edits API to delete server-side without invalidating the cached prefix.

Time-Based
Gap > 60 min. Keeps last 5 compactable results. Replaces older content with [Old tool result content cleared].
Cached-MC
Queues cache_edits blocks. Does NOT mutate local messages. Boundary message records actual cache_deleted_input_tokens after response.
Config defaults
gapThresholdMinutes: 60  ·  keepRecent: 5
Only runs on
Main thread (not subagents)
L3
Session-Memory Compact sessionMemoryCompact.ts
Trigger: tokens > 167K  ·  tries free summary before LLM
Zero Cost feature-gated

First layer inside autoCompactIfNeeded(). Uses the session memory file already extracted during the conversation — no new API call required. Returns null if post-compact count still exceeds threshold, falling through to Layer 4.

Prerequisites
tengu_session_memory + tengu_sm_compact flags enabled. Session memory file exists and is not just the template.
Keep window
minTokens: 10K  ·  minTextBlocks: 5  ·  maxTokens: 40K
Invariants
Never splits tool_use/tool_result pairs. Includes thinking blocks sharing message.id with kept messages. Never expands past last compact boundary.
Fallback
Returns null → Layer 4 fires
L4
Full LLM Compact compact.ts
Trigger: tokens > 167K & Layer 3 failed/unavailable
Full API Call

Makes an API call to summarize the conversation. Uses a forked agent (preferred) that reuses the main conversation's prompt cache. Falls back to direct streaming if the forked agent returns no text. Strips images and reinjected attachments before summarizing.

Forked agent
runForkedAgent() with querySource: 'compact'. maxTurns: 1, skipCacheWrite: true. Reuses main cache via identical cache-key params.
Output budget
COMPACT_MAX_OUTPUT_TOKENS: 20,000 tokens
PTL retry
Up to 3 retries. Drops oldest API round-trip groups (or 20%) each attempt.
Circuit breaker
After 3 consecutive failures, auto-compact is disabled for the session.
RC
Reactive Compact reactiveCompact.ts
Trigger: 413 Prompt-Too-Long error from API  ·  emergency recovery
API Call feature-gated

Safety net for cases where proactive layers fail or are disabled. Intercepted in query.ts when the API returns 413. Groups messages by API round-trip, peels groups from the tail, falls through to standard compaction. Context Collapse's recoverFromOverflow participates first.

Trigger
413 Prompt Too Long error intercepted in query.ts
Method
groupMessagesByApiRound() → peel tail groups → fall through to standard compact

Layer 2 Deep Dive

Microcompact Paths

Layer 2 has two mutually exclusive paths. The time-based path fires when the prompt cache has gone cold. The cached-MC path fires when the cache is warm — it deletes server-side without touching local message history.

Time-Based Microcompact
Cold cache  ·  mutates local messages  ·  timeBasedMCConfig.ts
Trigger Gap since last assistant message > 60 minutes
Threshold Matches 1-hour server prompt-cache TTL — cache guaranteed cold
Behavior Collect all compactable tool IDs in encounter order → keep last 5 → replace older content with [Old tool result content cleared]
keepRecent 5 (minimum 1)
Side effects Resets cached-MC state (cache invalidated anyway)
Compactable Tools
Read Shell Grep Glob WebSearch WebFetch Edit Write
Cached Microcompact
Warm cache  ·  server-side only  ·  ant-only, feature-gated
Trigger Count-based threshold via persistent CachedMCState
Mechanism Queues cache_edits blocks for the API layer — does NOT modify local message content
How it works Tracks tool results across turns. API receives cache_reference + cache_edits. Server deletes without invalidating cached prefix.
Accounting After API response, deferred boundary message records actual cache_deleted_input_tokens
Priority Takes precedence over time-based MC (mutually exclusive per turn)
Key Advantage
Preserves the prompt cache hit — the client never rewrites the cached prefix, so subsequent turns still benefit from cache reads. Time-based MC always pays full reprocessing cost.

Layer 4 Deep Dive

Full Compact Pipeline

The heavyweight path: forked agent summarization, 9-section prompt, PTL retry logic, and post-compact restoration. Click any step to see details or run the full animation.

1
Pre-Compact Hooks Fire
executePreCompactHooks() — registered hooks can inject custom instructions into the compact prompt or display a message to the user.
Hooks are registered by extensions and plugins. They receive a context object and can return additional instructions that get prepended to the compact prompt. They can also surface a status message in the UI while compaction runs.
2
Pre-Processing: Strip & Clean
stripImagesFromMessages() replaces image/document blocks with [image]/[document] markers. stripReinjectedAttachments() removes skill_discovery and skill_listing (re-surfaced post-compact anyway).
Stripping images prevents the summarization call itself from hitting prompt-too-long limits. Removing skill attachments avoids redundant token spend — those will be re-injected as part of post-compact restoration regardless.
3
streamCompactSummary() — Forked Agent
Preferred path: runForkedAgent() with querySource: 'compact'. Reuses main conversation's prompt cache by sending identical cache-key params. maxTurns: 1, skipCacheWrite: true.
The forked agent shares the cache with the main conversation thread, dramatically reducing cost. If the fork returns no text (e.g., model attempted a tool call), falls back to direct streaming via queryModelWithStreaming() with maxOutputTokens: 20,000.
4
9-Section Prompt Template
The LLM is instructed to produce a structured summary with 9 required sections.
1. Primary Request and Intent 2. Key Technical Concepts 3. Files and Code Sections 4. Errors and Fixes 5. Problem Solving 6. All User Messages ★ 7. Pending Tasks 8. Current Work 9. Optional Next Step

Section 6 is marked critical: "List ALL user messages that are not tool results. These are critical for understanding the users' feedback and changing intent." This preserves user corrections even when the conversation is truncated.

5
Analysis Scratchpad Stripped
The prompt asks for an <analysis> block before the <summary> block. formatCompactSummary() strips it — the scratchpad never enters the context window.
The analysis block acts as chain-of-thought drafting that improves summary quality without consuming tokens in the final output. Only the content inside <summary>...</summary> survives into the compressed history.
6
PTL Retry: truncateHeadForPTLRetry()
If the compact request itself hits prompt-too-long: group by API round, drop oldest groups (or 20%), prepend truncation marker, retry. Up to MAX_PTL_RETRIES = 3.
groupMessagesByApiRound() splits on assistant message ID boundaries — finer-grained than human turns. A synthetic "[earlier conversation truncated for compaction retry]" marker is prepended if the trimmed result starts with an assistant message (to preserve API invariants).
7
Post-Compact Restoration
Critical context re-injected: recently-read files (50K budget, 5K each), active plan, invoked skills (25K budget, 5K each), deferred tools delta, MCP instructions, SessionStart hooks.
Note: sentSkillNames is NOT reset — avoids re-injecting ~4K tokens of skill listing. The model still has SkillTool in its schema. Invoked skill content survives so createSkillAttachmentIfNeeded() can include full skill text in subsequent compaction attachments.
